# Secretary — MVP inicial (2026-07-13)

Resumen de lo implementado hoy: backend de gestión de recibos/gastos y una app
Expo para probarlo desde el celular.

## Backend (`backend/`)

API FastAPI de un solo usuario, sin autenticación, para llevar recibos y gastos.

- **Stack**: FastAPI, SQLAlchemy, MySQL (vía `pymysql`), Pydantic Settings.
- **Modelo `Receipt`**: vendor, amount, purchase_date, category, image_path,
  raw_ocr_text, timestamps.
- **Endpoints** (`/receipts`):
  - `POST /receipts` — alta manual (JSON).
  - `POST /receipts/upload` — sube una imagen, corre OCR con Google Vision y
    completa vendor/monto/fecha con heurísticas de texto; si no hay
    credenciales configuradas, guarda igual y deja constancia del error en
    `raw_ocr_text` en vez de romper.
  - `GET /receipts`, `GET /receipts/{id}`, `PATCH /receipts/{id}`,
    `DELETE /receipts/{id}`.
  - `GET /receipts/summary` — total gastado y desglose por categoría.
- **CORS** habilitado (`allow_origins=["*"]`) para poder pegarle desde la app
  Expo en desarrollo local.
- **MySQL local**: la instancia de Anaconda nunca había sido inicializada en
  esta máquina — se corrió `mysql_install_db`, se arrancó el servicio y se
  reseteó la password de `root` (ahora en `backend/.env`, gitignoreado). Se
  creó la base `secretary`.

## Frontend (`frontend/`)

App Expo (React Native) con TypeScript y `expo-router`, elegida porque el
objetivo a largo plazo del proyecto es una app mobile-first de uso personal.

- **Tabs**: `Receipts` (lista, alta, edición, borrado) y `Summary` (total y
  por categoría).
- **Alta de recibos**: cámara o galería (sube a `/receipts/upload`) o carga
  manual por formulario.
- **`src/lib/api.ts`**: cliente tipado contra el backend; resuelve
  automáticamente la IP de la compu de desarrollo vía
  `Constants.expoConfig.hostUri`, así no hay que hardcodear la IP de la red.
- **Versión de Expo**: se bajó de SDK 57 → 56 → 55 → **54** para que coincida
  con la versión de Expo Go disponible en el App Store al momento de probar
  (54.0.2). Se sacaron `expo-status-bar` y `expo-image` del array de
  `plugins` en `app.json` porque su resolución de config-plugin está rota en
  las versiones instaladas y ninguno de los dos necesita uno.
- Se corrigió un import roto en `_layout.tsx`: `ThemeProvider`, `DarkTheme` y
  `DefaultTheme` no se re-exportan desde `expo-router` en SDK ≤56 — vienen de
  `@react-navigation/native`.
